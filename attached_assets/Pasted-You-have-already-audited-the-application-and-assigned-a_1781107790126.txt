You have already audited the application and assigned a production readiness score.

Your previous task was to identify problems.

Your new task is different.

Your objective is to increase the production readiness score to at least 90/100 and ideally 95+/100 through meaningful improvements.

Do not focus on cosmetic changes.

Focus on improvements that increase real-world reliability, trustworthiness, security, scalability, maintainability, user retention, observability, and operational readiness.

## Step 1: Re-Evaluate

Before making changes:

* Recalculate the current production readiness score.
* Explain exactly why the score is not yet 90+.
* Identify every remaining weakness.
* Rank weaknesses by impact.

## Step 2: Missing Feature Discovery

Act as a senior product team.

Determine which important features are still missing.

Do not limit yourself to the existing specification.

Ask:

* What would experienced users expect?
* What would support teams need?
* What would administrators need?
* What would operations teams need?
* What would security teams need?
* What would developers maintaining this project need?

Identify features that should exist but currently do not.

## Step 3: Production Infrastructure

Evaluate whether the application has:

* monitoring
* health checks
* uptime monitoring
* error tracking
* performance monitoring
* audit logging
* security logging
* activity logs
* deployment safeguards
* backup procedures
* recovery procedures

Implement anything important that is missing.

## Step 4: Notification System

Implement a complete notification strategy.

Use Resend for email delivery where appropriate.

Evaluate and implement:

* account verification emails
* password reset emails
* login alerts
* security alerts
* reward notifications
* transaction notifications
* account change notifications
* referral notifications
* support notifications
* administrative alerts

Ensure email delivery failures are handled properly.

Implement templates, retries, logging, and monitoring.

## Step 5: User Trust Improvements

Identify anything that reduces user confidence.

Improve:

* onboarding
* messaging
* account transparency
* reward transparency
* transaction visibility
* notification clarity
* user feedback mechanisms

Users should always understand:

* what happened
* why it happened
* what happens next

## Step 6: Security Hardening

Assume the application becomes popular tomorrow.

Implement additional protections where appropriate:

* rate limiting
* abuse detection
* fraud detection
* anomaly detection
* suspicious activity monitoring
* account lockout protections
* session protection
* API protection
* audit trails

Attempt to improve security beyond minimum requirements.

## Step 7: Scalability Review

Assume:

* 100 users
* 1,000 users
* 10,000 users
* 100,000 users

Identify bottlenecks.

Improve architecture where necessary.

Document future scaling risks.

## Step 8: Failure Analysis

For every important workflow ask:

* What happens if the database fails?
* What happens if the API fails?
* What happens if email delivery fails?
* What happens if a third-party service fails?
* What happens if a request times out?
* What happens if duplicate requests occur?
* What happens if users attempt abuse?

Implement protections wherever possible.

## Step 9: Competitive Benchmarking

Compare the application against mature commercial products.

Identify:

* missing capabilities
* missing safeguards
* missing operational tooling
* missing user experience improvements

Implement the most valuable additions.

## Step 10: Final Optimization Pass

Search for:

* technical debt
* unfinished workflows
* hidden bugs
* inconsistent behavior
* weak error handling
* poor recovery paths

Refactor and improve them.

## Final Deliverables

Produce:

1. Updated production readiness score.
2. Features added.
3. Security improvements added.
4. Infrastructure improvements added.
5. User trust improvements added.
6. Scalability improvements added.
7. Remaining weaknesses.
8. Exact steps required to reach 100/100 readiness.
9. Final deployment recommendation.

Do not stop after finding issues.

Implement improvements wherever possible and re-evaluate until no high-impact opportunities remain.
